Вивчення Git

Що таке система контролю версій?

Система контролю версій (Version Control System, VCS) — це інструмент, який дозволяє зберігати історію змін у проєктах, працювати командно та легко повертатися до попередніх версій коду.

Git — найпопулярніша система контролю версій, розроблена Лінусом Торвальдсом у 2005 році. Вона є розподіленою, тобто кожен учасник має повну копію репозиторію.

Переваги Git

  • Збереження історії змін у проєкті
  • Можливість командної роботи над одним кодом
  • Швидке перемикання між версіями
  • Робота без постійного підключення до інтернету
  • Велика спільнота та підтримка

10 базових команд Git

  1. git init – створити новий репозиторій
  2. git clone URL – клонувати репозиторій з GitHub
  3. git status – перевірити стан файлів
  4. git add filename – додати файл у staging
  5. git commit -m "повідомлення" – зафіксувати зміни
  6. git log – переглянути історію комітів
  7. git branch – показати гілки
  8. git checkout branch_name – переключитися на іншу гілку
  9. git merge branch_name – об’єднати гілки
  10. git push – відправити зміни на GitHub
